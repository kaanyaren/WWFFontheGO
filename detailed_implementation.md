## 2. State Management & Routing
- **Riverpod (`hooks_riverpod`)**: Used as the primary dependency injection and reactive state management solution. Features will utilize `AsyncNotifierProvider` for operations like writing to Isar or fetching from Firebase.
- **GoRouter**: Will handle deep linking and programmatic navigation.
  - `/` -> Main Dashboard (Holds the `animated_notch_bottom_bar`)
  - `/log` -> QSO Log List -> Sub-route `/log/new` for the entry form.
  - `/spots` -> Live Spots view.
  - `/parks` -> Firebase Park List view.
  - `/settings` -> User preferences.

## 3. Local Database Schema (Isar)
The app is entirely **offline-first**. All critical data resides in local Isar collections.

### QsoLog Collection
Maps directly to the ADIF specification required by WWFF.
- `id` (Isar.autoIncrement)
- `callsign` (String, Indexed) - Hunter's callsign
- `qsoDate` (DateTime, Indexed) - Derived to standard YYYYMMDD
- `timeOn` (String) - Derived to HHMMSS
- `band` (String, Indexed) - E.g., '20m'
- `mode` (String, Indexed) - E.g., 'SSB'
- `stationCallsign` (String) - The activator's sign
- `operatorCallsign` (String)
- `mySig` (String) - Fixed to 'WWFF'
- `mySigInfo` (String, Indexed) - Activator's park (e.g. TAFF-0123)
- `sig` (String?) - 'WWFF' for P2P
- `sigInfo` (String?) - Hunter's park for P2P
- `rstSent` & `rstRcvd` (String)

### UserProfile Collection (Singleton)
- `id` (Fixed: 1)
- `defaultCallsign` (String)
- `defaultOperator` (String)
- `defaultMySigInfo` (String) - Represents the active park the user is at.
- `defaultTxPower` (int)

## 4. External Integrations
### 4.1 WWFF Spots (Live JSON parsing)
- **Source**: `https://wwff.co/static/spots.json`
- **Method**: Polled via `Dio` every 60 seconds when on the Spots viewing page. The parsed JSON will be converted into Dart objects and fed into a Riverpod `FutureProvider`. Users can tap a spot to pre-fill a new `QsoLog`.

### 4.2 Firebase Park References
- **Source**: Firestore (Read-Only access).
- **Update Mechanism**: A GitHub Action running daily will scrape wwff.co or update the database with the latest park references and photo URLs. The Flutter app will fetch this data and cache it locally in Isar to ensure the lists work offline in the wilderness.

## 5. UI/UX Elements
- **Theme**: "Organic Glassmorphism". Card widgets will use `color.withOpacity(0.85)` over a nature-tinted scaffold (`#F4F9F4`).
- **Typography**: Google Fonts `Geist` with varying weights to prioritize data readability (high-contrast data cards).
- **Navigation**: `animated_notch_bottom_bar` to provide fluid, modern switching between Logging, Spots, and Settings.

## 6. Import/Export
- **ADIF Export**: Critical for WWFF submissions. The utility will parse the `QsoLog` collection into a `.adif` plaintext string, matching the strict WWFF requirements, and output it via `share_plus` to allow users to send it to their emails or files.
- **JSON Backup**: Profile settings and favorite parks will be exportable as pure JSON objects.
